# Password Reset Flow Wireframe

## Page Purpose
Allows users to securely reset their password when they've forgotten it, using email-based verification and a secure link.

## URL Patterns
- Request reset: `/forgot-password`
- Reset page: `/reset-password?token={token}`

## Flow Steps
1. Request Reset - User enters email
2. Email Sent - Confirmation message
3. Reset Page - Create new password (from email link)
4. Success - Password updated

## Component Specifications

### Password Input with Toggle
- Height: 48px
- Border: 1px solid #E5E7EB (default), #0D6EFD (focused), #DC3545 (error)
- Border radius: 8px
- Eye icon: Toggle between show/hide
- Font: System font, 16px

### Password Strength Indicator
- Bar height: 4px
- Segments: 4 (Weak/Fair/Good/Strong)
- Colors: Red (#DC3545), Yellow (#FFC107), Light Green (#20C997), Green (#198754)
- Labels: Weak, Fair, Good, Strong

### Requirement Checklist
- Check icon: Green (#198754) when met
- X icon: Red (#DC3545) when not met
- Circle icon: Gray (#6C757D) when not yet evaluated
- Real-time validation as user types

## Accessibility

### ARIA Labels
- Password inputs: `aria-describedby` pointing to requirements list
- Show/hide toggle: `aria-label="Show password"` / `aria-label="Hide password"`
- Requirement items: `aria-live="polite"` for status changes
- Success/Error messages: `role="alert"`

### Keyboard Navigation
- Tab through form fields
- Enter to submit
- Space to toggle show/hide
- Focus visible on all interactive elements

### Screen Reader
- Announce password requirements as they're met/unmet
- Announce password strength changes
- Clear error announcements
- Success confirmation announcement

### Color + Icon
- Never rely on color alone
- Check/X icons accompany requirement status
- Error messages have icon + text

---

## Security Considerations

### Token Security
- Single-use tokens
- 24-hour expiration
- Random, cryptographically secure
- Invalidate previous tokens when new one requested

### Rate Limiting
- Max 3 reset requests per email per hour
- Block after 5 failed reset attempts
- Log all reset attempts

### Password Policy
- Minimum 8 characters
- At least 1 uppercase letter
- At least 1 number
- At least 1 special character (recommended)
- Check against common password lists
- Prevent reuse of last 3 passwords

### Confirmation Behavior
- Generic "if account exists" message (prevent enumeration)
- Always show success even if email doesn't exist
- Send email to existing accounts notifying of reset attempt
