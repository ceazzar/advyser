# Email/Phone Verification Wireframe

## Page Purpose
Allows users to verify their email address or phone number to complete account registration, or verify phone control during business claim process.

## URL Patterns
- Email verification: `/verify/email?token={token}`
- Phone verification: `/verify/phone`
- Claim verification: `/claim/{business-slug}/verify`

## Variants
1. Email Verification (link-based)
2. Phone Verification (OTP-based)
3. Claim Proof-of-Control (OTP to business phone)

## Component Specifications

### OTP Input Box
- Size: 48x56px (desktop), 40x48px (mobile)
- Border: 2px solid #E5E7EB (default), #0D6EFD (focused), #DC3545 (error)
- Border radius: 8px
- Font: 24px, center-aligned, monospace
- Auto-advance: Move focus to next box on input
- Backspace: Clear current, move to previous if empty
- Paste support: Distribute digits across all boxes

### Countdown Timer
- Format: "Resend in 0:47"
- Duration: 60 seconds default
- Visual: Gray text, disabled button state
- Completion: Enable button, change to "Resend Code" link

### Verification Icons
- Size: 64x64px
- Email: Envelope icon
- Phone: Mobile phone icon
- Success: Green checkmark in circle
- Error: Red X in circle

## Accessibility

### ARIA Labels
- OTP inputs: `aria-label="Digit X of 6"`, grouped with `role="group"`
- Countdown: `aria-live="polite"` for timer updates
- Success/Error: `role="alert"` for state changes
- Resend button: `aria-disabled="true"` during cooldown

### Keyboard Navigation
- Tab: Move between OTP boxes, resend button, change link
- Arrow keys: Navigate between OTP boxes
- Backspace: Clear current digit, move to previous
- Enter: Submit when all digits filled
- Paste: Auto-fill from clipboard (Ctrl/Cmd+V)

### Focus States
- OTP boxes: Bold border, subtle glow
- Buttons: Visible focus ring
- Links: Underline + focus ring

### Screen Reader
- Announce: "Enter 6-digit verification code"
- On input: "Digit 1 entered" (or similar)
- On error: "Invalid code. Please try again. 2 attempts remaining."
- On success: "Verification successful. Phone verified."

---

## Technical Notes

### OTP Auto-Submit
- Trigger verification automatically when 6 digits entered
- Show loading state immediately
- Clear inputs on error for retry

### Rate Limiting (Backend)
- Max 3 resend requests per 10 minutes
- Max 5 verification attempts per code
- Lockout after 10 failed attempts (15 min cooldown)

### Code Expiry
- Email links: 24 hours
- Phone OTP: 10 minutes
- Claim OTP: 10 minutes

### Security
- Partially mask email: j***@email.com
- Partially mask phone: +61 412 XXX 789
- Never reveal full contact info on page
- Log verification attempts for audit
