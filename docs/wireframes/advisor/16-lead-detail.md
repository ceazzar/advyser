## Components

### Header Section
- **Back Link**: "â† Back to Leads" - returns to leads pipeline
- **Consumer Name**: First name + last initial (or "Anonymous" if hidden)
- **Status Dropdown**: Current status with option to change
- **Convert to Client Button**: Primary CTA when ready to convert

### Status Actions Bar
| Status | Available Actions |
|--------|-------------------|
| New | Accept, Decline, Request More Info |
| Contacted | Mark as Booked, Request More Info |
| Booked | (Actions in booking section) |
| Converted | (Read only) |
| Declined | (Read only) |

### Client Brief Card
| Field | Description |
|-------|-------------|
| Goal | Full description of what consumer is looking for |
| Timeline | When they want to proceed |
| Budget | Their budget range |
| Meeting Preference | Video / Phone / In-person with icon |
| Location | City, State, Postcode |
| Documents | List of uploaded files with download links |

### Internal Notes Section
- Textarea for adding new notes
- Privacy notice: "Notes are private and not visible to the consumer"
- List of existing notes with:
  - Author name
  - Timestamp
  - Note content
  - Edit/Delete actions (for own notes only)

### Activity Timeline
Chronological list of all activities:
| Event Type | Description |
|------------|-------------|
| Lead received | When lead was created |
| Status change | When status was updated |
| Note added | When internal note was added |
| Message sent | When message was sent to consumer |
| Message received | When consumer replied |
| Booking created | When meeting was scheduled |
| Document uploaded | When consumer added document |

### Quick Actions Sidebar
| Action | Description |
|--------|-------------|
| Send Message | Opens message thread with consumer |
| Create Booking | Opens booking modal |
| View Profile | Opens consumer's profile (if available) |
| Generate Report (Pro) | Opens Copilot report generator |

## Accessibility

- Back link announces navigation destination
- Status dropdown is keyboard accessible
- Action buttons have clear focus states
- Activity timeline uses semantic list markup
- Document download links include file size for screen readers
- Notes section properly labeled for screen readers
- Modals trap focus and can be closed with Escape key
