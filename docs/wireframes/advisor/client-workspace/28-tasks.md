# Client Workspace - Tasks Tab Wireframe

## Page Purpose
The Tasks tab provides a focused task management area for all to-dos related to a specific client. Tasks are separated into "Advisor Tasks" (your responsibilities) and "Client Tasks" (waiting on the client), with support for manual creation and Copilot-generated tasks from meeting action items.

## Business Logic

- Overdue calculation: Task is overdue if `dueDate < today` and `status === 'pending'`
- Due today: Task due date matches current date
- Tasks without due dates appear at end of list (unless reordered)
- Completing a task auto-fills `completedAt` timestamp
- Uncompleting clears `completedAt` and returns to `pending`
- Copilot-generated tasks link back to the Copilot run they came from
- Client tasks are for tracking what you're waiting on; they don't notify the client

## Accessibility Notes

- Checkbox has proper label association
- Drag handle has aria-label "Reorder task"
- Drag and drop has keyboard support (spacebar to grab, arrows to move)
- Task sections are landmark regions
- Overdue indicator announced by screen readers
- Delete confirmation is modal dialog with focus trap
- Filter changes announced via aria-live region
- Completed tasks have aria-checked="true"
