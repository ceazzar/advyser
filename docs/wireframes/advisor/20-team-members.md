## API Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `GET /api/advisor/team` | GET | List team members |
| `POST /api/advisor/team/invite` | POST | Send invitation |
| `GET /api/advisor/team/invitations` | GET | List pending invitations |
| `DELETE /api/advisor/team/invitations/:id` | DELETE | Cancel invitation |
| `POST /api/advisor/team/invitations/:id/resend` | POST | Resend invitation |
| `PATCH /api/advisor/team/:id` | PATCH | Update member role |
| `DELETE /api/advisor/team/:id` | DELETE | Remove member |
| `POST /api/advisor/team/transfer` | POST | Transfer ownership |
| `GET /api/advisor/team/permissions` | GET | Get role permissions |

### TeamMember
```typescript
interface TeamMember {
  id: string;
  userId: string;
  advisorAccountId: string;
  email: string;
  name: string;
  avatar?: string;
  role: 'owner' | 'admin' | 'staff';
  status: 'active' | 'inactive';
  permissions: Permission[];
  lastActiveAt: Date;
  joinedAt: Date;
}

interface TeamInvitation {
  id: string;
  advisorAccountId: string;
  email: string;
  role: 'admin' | 'staff';
  permissions: Permission[];
  message?: string;
  invitedBy: string;
  sentAt: Date;
  expiresAt: Date;
  status: 'pending' | 'accepted' | 'expired' | 'cancelled';
}

type Permission =
  | 'view_all_leads'
  | 'view_assigned_leads'
  | 'accept_decline_leads'
  | 'send_messages'
  | 'manage_bookings'
  | 'view_analytics'
  | 'edit_profile'
  | 'manage_services'
  | 'manage_team'
  | 'manage_billing';
```

## Invitation Flow

1. Owner/Admin clicks "Invite Team Member"
2. Enter email and select role
3. Optionally customize permissions and add message
4. System sends invitation email
5. Invitee receives email with accept link
6. Invitee creates account or logs in
7. Invitee is added to team with selected role
8. Owner sees new member in Active Members list
