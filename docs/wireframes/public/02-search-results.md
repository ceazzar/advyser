# Search Results Page Wireframe

## Page Purpose
Display a filterable, sortable list of advisors matching the user's search criteria, enabling consumers to compare advisors and select one to view in detail or contact directly.

## URL Pattern
`/search?location={location}&type={advisor_type}&specialty={specialty}&radius={km}&sort={sort_by}`

## User Role
public (works for both authenticated and unauthenticated users)

## Components

### Search Bar (Pre-filled)
- Type: `form/search`
- Data fields:
  - Location (pre-filled from query param)
  - Advisor type (pre-filled from query param)
- Actions:
  - [Search] - re-executes search with new params
- States:
  - Default (values populated)
  - Editing (user modifying)

### Filters Sidebar
- Type: `form/filters`
- Data fields:
  - Location radius (radio group)
  - Advisor type (radio group)
  - Specialty (checkbox group, multi-select)
  - Service mode (checkbox group)
  - Fee model (checkbox group)
  - Verified only (toggle)
  - Available this week (toggle)
- Actions:
  - Any filter change - updates URL params, re-fetches results
  - [Clear All Filters] - resets all filters
  - [Show more] - expands collapsed specialty options
- States:
  - Default
  - Filter active (highlighted)
  - Collapsed (mobile - shown as modal)

### Results Header
- Type: `section/header`
- Data fields:
  - Result count
  - Location display
  - Active filters (pills with remove)
  - Sort dropdown
- Actions:
  - [Ã—] on filter pill - removes that filter
  - Sort dropdown change - updates sort param
- States:
  - Results found
  - No results (shows empty state)
  - Loading (shows skeleton)

### Advisor Result Card
- Type: `card/advisor`
- Data fields:
  - `avatar_url` (string)
  - `name` (string)
  - `business_name` (string)
  - `advisor_type` (enum)
  - `location` (suburb, state)
  - `distance_km` (number, calculated)
  - `verified` (boolean)
  - `specialties` (array, show top 3)
  - `price_tier` (1-4, shown as $ symbols)
  - `response_time` (string)
  - `rating` (number)
  - `review_count` (number)
- Actions:
  - [View Profile] - navigates to `/advisor/{slug}`
  - [Request Intro] - opens request intro modal/page
  - Click card - navigates to profile
- States:
  - Default
  - Hover (elevated shadow)
  - Loading (skeleton)

### Sort Dropdown
- Type: `dropdown/select`
- Data fields:
  - Selected option
  - Options: Recommended, Nearest, Highest rated, Soonest availability
- Actions:
  - Select option - updates URL param, re-fetches
- States:
  - Closed
  - Open

### Pagination

```

radius: number (km, default 25)
type: string (advisor_type enum)
specialty: string[] (array of specialties)
service_mode: string[] (in_person, online, phone)
fee_model: string[] (fee_for_service, commission, hybrid, flat)
verified_only: boolean
available_this_week: boolean
sort: string (recommended, nearest, highest_rated, soonest_availability)
page: number (default 1)
limit: number (default 10)
```

## AU Compliance Notes

1. **Verified Badge Transparency**
   - Verified badge should have tooltip explaining what it means
   - "AFSL/Licence verified via ASIC register"

2. **No Endorsement Language**
   - "Recommended" sort is based on algorithm (response time, completeness, activity)
   - Not an endorsement of quality of advice

3. **Price Tier Disclaimer**
   - Price indicators ($-$$$$) are estimates only
   - Actual fees determined by advisor

4. **Review Source**
   - Reviews are from verified platform engagements only
   - No imported reviews from other sources
