## Components

### Email Input
- Type: `form/input`
- Data fields:
  - `email` (string)
- Actions:
  - Input - updates value
  - Blur - basic format validation
- States:
  - Default
  - Focused
  - Error (invalid format or failed auth)

### Password Input
- Type: `form/password`
- Data fields:
  - `password` (string)
  - `show_password` (boolean)
- Actions:
  - Input - updates value
  - Click [ğŸ‘] - toggles visibility
- States:
  - Default
  - Focused
  - Hidden (dots)
  - Visible (plain text)
  - Error (failed auth)

### Login Button
- Type: `button/submit`
- Data fields: None
- Actions:
  - Click - validates and submits form
- States:
  - Default
  - Hover
  - Loading (spinner)
  - Disabled (form incomplete)

### OAuth Button (Google)
- Type: `button/oauth`
- Data fields: None
- Actions:
  - Click - initiates Google OAuth flow
- States:
  - Default
  - Hover
  - Loading

### Forgot Password Link
- Type: `link/text`
- Data fields: None
- Actions:
  - Click - opens forgot password modal/view
- States:
  - Default
  - Hover

### Sign Up Link
- Type: `link/text`
- Data fields: None
- Actions:
  - Click - navigates to /signup
- States:
  - Default
  - Hover

### Error Alert
- Type: `alert/error`
- Data fields:
  - `message` (string)
- Actions: None (display only)
- States:
  - Hidden (no error)
  - Visible (error occurred)

### Forgot Password Modal
- Type: `modal/form`
- Data fields:
  - `email` (string, pre-filled if entered in login form)
- Actions:
  - Click [Send Reset Link] - sends reset email
  - Click [Ã—] or [â† Back to Login] - closes modal
- States:
  - Form view
  - Loading
  - Success view

## Interactions

| Trigger | Action |
|---------|--------|
| Type in email field | Update email value |
| Type in password field | Update password value |
| Click [ğŸ‘] | Toggle password visibility |
| Click [Log In] | Validate form, attempt login |
| Press Enter (in form) | Submit login form |

| Method | Endpoint | Description |

| Field | Rules |

## Security Considerations

1. **Rate Limiting**
   - Login attempts limited to 5 per minute per IP
   - Forgot password limited to 3 per hour per email

2. **Error Messages**
   - Generic "Invalid email or password" (don't reveal if email exists)
   - Forgot password always says "If an account exists..." (don't confirm email existence)

3. **Password Reset**
   - Token expires in 1 hour
   - Single use token
   - Invalidates previous reset tokens

4. **Session Management**
   - JWT token with short expiry (1 hour)
   - Refresh token with longer expiry (7 days)
   - Secure, httpOnly cookies

5. **Brute Force Protection**
   - Progressive delays after failed attempts
   - CAPTCHA after 3 failed attempts
   - Account lockout after 10 failed attempts (30 min)

## AU Compliance Notes

1. **Privacy**
   - Login page does not track or store passwords in logs
   - Failed attempts logged without password content

2. **Security Standards**
   - Password transmission over HTTPS only
   - No password hints or security questions

3. **Data Retention**
   - Login activity logged for security (IP, timestamp, success/fail)
   - Retained for 12 months per AU data retention guidelines
