## Components

### Conversation Header
- Type: header bar
- Data fields:
  - Advisor avatar
  - Advisor name
  - Business name
  - Status indicator (Online/Away/Offline)
  - View Profile link

### Context Bar
- Type: info banner
- Data fields: Request type/service
- Actions: Click "View Request" â†’ Navigate to request detail
- States: default

### Message Thread
- Type: scrollable message list
- Data fields: Messages grouped by date
- Auto-scroll: Scrolls to latest message on load
- States: loading, empty, default

### Message Bubble (User)
- Type: chat bubble
- Position: Right-aligned, blue background
- Data fields: Message content, timestamp

### Message Bubble (Advisor)
- Type: chat bubble
- Position: Left-aligned, gray background

- States: default

### Date Separator
- Type: divider with text
- Data fields: Date string (e.g., "Monday, 15 January 2024")
- Format: "Today", "Yesterday", or full date
- States: static

### Attachment Display
- Type: file card
- Data fields: File icon, filename, file size

- Max size displayed: e.g., "2.4 MB"
- States: default, downloading, downloaded

### Typing Indicator
- Type: indicator bubble
- Position: Left-aligned (shows advisor is typing)
- Animation: Three animated dots
- States: visible, hidden

### Message Input
- Type: textarea with actions
- Placeholder: "Type your message..."
- Max length: 5000 characters

- Max file size: 10MB

### Send Button

## API Endpoints

- `GET /api/consumer/conversations/{id}` - Fetch conversation details and metadata
- `GET /api/consumer/conversations/{id}/messages` - Fetch messages (paginated)
  - Query params: `before=messageId`, `limit=50`
- `POST /api/consumer/conversations/{id}/messages` - Send a new message
- `POST /api/consumer/conversations/{id}/messages/attachments` - Upload attachment
- `PUT /api/consumer/conversations/{id}/read` - Mark conversation as read
- `GET /api/consumer/conversations/{id}/messages/stream` - WebSocket for real-time messages

## Request Body (Send Message)

```json
{
  "content": "Hi John, thank you for getting back to me!",
  "attachments": []
}
```

## WebSocket Events

- `message:new` - New message received
- `message:read` - Message read receipt
- `typing:start` - User started typing
- `typing:stop` - User stopped typing
- `status:change` - Advisor online status changed

## Accessibility

- Message thread has proper ARIA live region for new messages
- Keyboard navigation through messages with arrow keys
- Send button has aria-label "Send message"
- Attachment button has aria-label "Attach file"
- Read receipts announced to screen readers
- Typing indicator announced when visible
- Date separators have proper heading roles
- File download links are keyboard accessible
