## Components

### Header Section
- Type: page header
- Data fields: Advisor name, business name, status badge, submission date
- Actions: Back link â†’ Return to requests list
- States: default

### Status Timeline
- Type: progress timeline
- Data fields: Steps (Submitted, Advisor Viewed, Responded, Booked, Completed), timestamps
- Variants:
  - Completed step (â—) - solid dot, checkmark
  - Current step - highlighted
  - Pending step (â—‹) - outline dot
- States: varies by request status

### Your Brief Section
- Type: content card
- Data fields: Goal, Timeline, Budget, Preferences (bullet list)
- Actions: None (read-only display of submitted information)
- States: default

### Advisor Response Section
- Type: message card
- Data fields: Advisor avatar, name, business, message content, timestamp, attachments
- Actions: Download attachment
- States: pending (waiting message), received (with content)

### Advisor Sidebar Card
- Type: profile card
- Data fields: Avatar, name, business, rating, review count, advisor type, location, email, phone
- Actions: View Full Profile â†’ Navigate to advisor profile, Send Message, Book Appointment
- States: default

### Action Buttons
- Type: button group
- Variants by status:
  - New/Contacted: [Send Message] [Book Appointment]
  - Booked: [Send Message] [View Booking]
  - Completed: [Send Message] [Leave Review]
- States: default, hover, disabled

### Attachment Display
- Type: file attachment
- Data fields: File icon, filename, file size
- Actions: Download
- States: default, downloading

## API Endpoints

- `GET /api/consumer/requests/{id}` - Fetch request detail
- `GET /api/consumer/requests/{id}/timeline` - Fetch status timeline
- `GET /api/consumer/requests/{id}/response` - Fetch advisor response
- `POST /api/consumer/requests/{id}/cancel` - Cancel request
- `GET /api/advisors/{id}/contact` - Fetch advisor contact info
