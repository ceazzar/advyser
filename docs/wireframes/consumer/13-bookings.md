## Components

### Tab Navigation
- Type: tab bar
- Tabs: Upcoming, Past
- Data fields: Tab label, count badge
- Actions: Click tab â†’ Filter bookings by timing
- States: active, inactive

### View Toggle
- Type: toggle button group
- Options: List View, Calendar View
- Actions: Click â†’ Switch between list and calendar display
- States: active, inactive

### Booking Card
- Type: card with two-column layout
- Left column data fields:
  - Advisor avatar
  - Advisor name
  - Business name
  - Advisor type
  - Meeting purpose/topic
  - Status badge
- Right column data fields:
  - Date (formatted: "Tomorrow", "Friday, 19 Jan")
  - Time range
  - Duration
  - Mode (Online/In-person)
  - Location or "Video Call"
- Actions: Join Meeting, View Details, Add to Calendar, Cancel Booking
- States: confirmed, pending, completed, cancelled

### Status Badge
- Type: badge/chip
- Variants:
  - Confirmed (green with checkmark) - Booking confirmed by advisor
  - Pending (yellow with clock) - Awaiting advisor confirmation
  - Completed (gray with checkmark) - Past booking
  - Cancelled (red with X) - Booking was cancelled
- States: static

### Join Meeting Button
- Type: primary button
- Visibility: Only for online meetings, within 15 min of start time
- Actions: Click â†’ Opens video meeting in new tab
- States: disabled (not yet time), enabled, loading

### Add to Calendar Button
- Type: dropdown button
- Options: Google Calendar, Apple Calendar, Outlook, Download .ics
- Actions: Select option â†’ Opens calendar or downloads file
- States: default, open

### Calendar View
- Type: month calendar grid
- Data fields: Dates with booking indicators
- Actions: Click date â†’ Shows booking details for that date
- Navigation: Previous/Next month arrows, Today button
- States: default, loading

## API Endpoints

- `GET /api/consumer/bookings` - Fetch all bookings
  - Query params: `status=upcoming|past`, `page=1`, `limit=10`
- `GET /api/consumer/bookings/{id}` - Fetch single booking detail
- `GET /api/consumer/bookings/calendar` - Fetch bookings in calendar format
  - Query params: `month=2024-01`
- `DELETE /api/consumer/bookings/{id}` - Cancel a booking
- `GET /api/consumer/bookings/{id}/meeting-link` - Get video meeting URL
- `GET /api/consumer/bookings/{id}/ics` - Download .ics file
